Consensus Algorithm

Miraila adopts the BFT consensus mechanism based on DPoS+VRF algorithm. DPoS uses the decentralized voting mechanism to conduct democratization of super nodes. At the same time, DPoS blocks can accommodate more transactions than PoW or PoS, greatly improving the number of transactions. The speed of the transaction. The BFT consensus mechanism is adopted between the super nodes, and more than 2/3 of the confirmation can achieve irreversible verification.

DAG

In order to solve the problem that only one main chain in the traditional blockchain can not be executed in parallel and the storage cost is high, Miraila uses the DAG ledger structure to store data. The DAG network structure has no transaction cost, and the transaction confirmation time is short, which is very suitable for storage. Large amounts of data.

Cross-chain

In the future, traditional Internet and blockchain applications will coexist. Miraila uses multi-signature technology smart contracts to achieve cross-chain transactions. Adapters create multi-signal addresses for the main chain, account lock, adaptable Bitcoin, Ethereum And other mainstream cryptocurrencies.

Smart Contract

By default, Miraila implements smart contracts using Docker containerization technology. In the industry, Docker as a virtual contract virtual machine has been applied in HyperLedger Fabric and has been demonstrated by a large number of enterprises and projects.
